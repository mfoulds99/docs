title: "Patron API Flow"
---

The Patron API flow describes the steps that occur when a patron (customer) places an order using the POS mobile app.

In this flow, the client is the mobile app used by the patron to order from the restaurant.

## 1. Select a Menu Category

When the patron selects a menu category in the mobile app, the client requests a list of dishes for the selected category. The server returns an array with a list of dishes. The client stores this information for future use.

<Info>
  The client uses this data to determine the appropriate dishes for the category. The dishes are not displayed in the mobile app immediately; the client must first check if there are sufficient ingredients in-stock for each dish. For details, see "2. Check Ingredient Quantities".
</Info>

### Endpoint

`GET /dishes`

### Query Parameters

**filter (string)**\
Filter the results to return the dishes for a specific category. Possible category values are `Appetizers`, `Main Courses`, and `Desserts`.

For example, use the following filter to return dishes for the `Appetizer` category:\
`filter=category.eq~Appetizer`.

**fields (string)**\
Specify the fields to return in the response. Provide a list of field names, separated by the `|` delimiter.

At a minimum, include the following fields in the response: `id`, `name`, `description`, `price`, `image_name`.

For example:\
`fields=id|name|description|price|image_name`

### Sample Request

```json
curl -X GET "{BaseURL}?filter=category.eq~Appetizer&fields=id|name|description|price|image_name"
```

## 2. Check Ingredient Quantities

Check the quantity of ingredients that are currently in-stock. The server returns an array with a list of all ingredients and their quantities.

The client uses this information to determine if there are sufficient ingredients for the dishes  before displaying them in the mobile app. Only dishes with essential ingredients greater than zero are displayed and can be selected by the patron.

### Endpoint

`GET /ingredients`

### Query Parameters

**fields (string)**\
Specify the fields to return in the response. Provide a list of field names, separated by the `|` delimiter. At a minimum, include the `name` and `in_stock_qty` parameters.

For example:\
`fields=name|in_stock_qty`

### Sample Request

```json
curl -X GET "{BaseURL}/ingredients?fields=name|in_stock_qty"
```

## 3. View Dish Details

View details of the dish selected by the patron. The server returns an array with a list of dishes and their details, such the the dish ID, name, price, and ingredients.

### Endpoint

`GET /dishes/{id}`

### Path Parameters

**id (string)**

Unique ID for the dish.

### Sample Request

```json
curl -X GET "{BaseURL}/dishes/698d4ef070b67b7e0cc0de24"
```

## 4. Add Dishes and View Cart

In the mobile app, the patron selects the dish they wish to order and adds the dish to their cart. The patron can add multiple dishes to their cart. This information is stored by the client for future use.

## 5. Place Order

In the mobile app, the patron clicks the "Order Now" button to place their order. The client requests a new order and sends information for the dishes in the patron's cart in the request body. The server creates a new order and returns the order details, including a unique ID for the order, the order status, and the order priority.

### Endpoint

`POST /orders`

### Request Body Parameters

**name (string) Required**\
Patron's name.

**table\_number (integer)**\
Patron's table number.

**dish\_ids (array of strings) Required**\
Array of the dish IDs in the order.

**special\_requests (string)**\
Any requests or instructions from the patron.

### Sample Request

```json
curl -H "Accept: application/json" -H "Authorization: Bearer 4840de21-ed92-44d9-8571-fac25e81c209" -X POST "{BaseURL}/orders" -H "Content-Type: application/json" -d '{"name": "James", "table_number": 10, "dish_ids": ["698d4ef070b67b7e0cc0de94", "698d4ef070b67b7e0cc0de24"], "special_requests": "no onions"}'
```

### Sample Response

```json
{
    "id": "69e525703825068c518b54a3",
    "created_at": "2026-04-19T18:56:48.518Z",
    "updated_at": "2026-04-19T18:56:48.518Z",
    "scheduled_at": null,
    "table_number": 10,
    "status": "Received",
    "priority": 3,
    "dish_ids": [
        "698d4ef070b67b7e0cc0de94",
        "698d4ef070b67b7e0cc0de24"
    ],
    "special_requests": "No onions"
}
```
