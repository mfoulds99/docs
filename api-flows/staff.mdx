title: "Staff API Flow"
---

The Staff API flow describes the steps that occur when kitchen staff receive an order placed by a patron.

In this flow, the client is the expeditor KDS app used by kitchen staff.

## Order Display

View a list of new orders placed on the current day. The server returns a list of all new orders in the system, which are then displayed in the KDS app.

### Endpoint

`GET /orders`

### Query Parameters

**filter (string)**\
Filter the results to only view orders with a `status` of `Received` and where the `created_at` date is the current date.

For example:\
`filter=status.eq~Received|created_at.gte2025-01-01`

### Sample Request

```json
curl -X GET "{BaseURL}/orders?filter=status.eq~Received|created_at.gte2025-01-01"
```

## Prioritization of Orders

Kitchen staff can use the KDS app to adjust an order's priority. The priority determines which orders should be filled first. The server updates the order priority and returns the order details to the client.

### Endpoint

`PUT /orders/{id}`

### Path Parameters

**id**\
Order ID.

### Request Body Parameters

**priority**\
Order priority. Valid values are 1-5 (1 is the highest priority).

**name**\
Patron's name.

**table\_number**\
Patron's table number.

**dish\_ids**\
Array of IDs for the dishes included in the order.

### Sample Request

```json
curl -H "Accept: application/json" -H "Authorization: Bearer 4840de21-ed92-44d9-8571-fac25e81c209" -X PUT "{BaseURL}/orders/698d4ef070b67b7e0cc0dec9" -H "Content-Type: application/json" -d '{"name": "James", "table_number": 10, "dish_ids": ["98d4ef070b67b7e0cc0de1f"], "priority": 1}'
```

### Sample Response

```json
{
    "id": "698d4ef070b67b7e0cc0dec9",
    "created_at": "2026-03-16T16:06:51.160Z",
    "updated_at": "2026-04-19T20:09:15.350Z",
    "scheduled_at": null,
    "table_number": 10,
    "status": "Received",
    "priority": 1,
    "dish_ids": [
        "698d4ef070b67b7e0cc0de1f"
    ],
    "special_requests": "no onions"
}
```

## Initiating Preparation

When ready to start an order, the kitchen staff use the KDS app to set the order status to `In Progress`. The server updates the order status and returns the order details to the client.

### Endpoint

`PUT /orders/{id}`

### Path Parameters

**id**\
Order ID.

### Request Body Parameters

**status**\
Order status. To initiate an order, set the status to `In Progress`.

**priority**\
Order priority. Valid values are 1-5 (1 is the highest priority).

**name**\
Patron's name.

**table\_number**\
Patron's table number.

**dish\_ids**\
Array of IDs for the dishes included in the order.

### Sample Request

```json
curl -H "Accept: application/json" -H "Authorization: Bearer 4840de21-ed92-44d9-8571-fac25e81c209" -X PUT "{BaseURL}/orders/698d4ef070b67b7e0cc0dec9" -H "Content-Type: application/json" -d '{"name": "James", "table_number": 10, "dish_ids": ["98d4ef070b67b7e0cc0de1f"], "priority": 1, "status": "In Progress"}'
```

## Order Preparation by Stations

Each dish in the order is assigned to a station. The kitchen staff use the KDS app to view the dishes assigned to their station.

The server returns an array with a list of dishes assigned to the requested station. The client stores this information for future use.

### Endpoint

`GET /orders/{id}/dishes`

### Path Parameters

**id**\
Order ID.

### Query Parameters

**filter (string)**\
Filter the results to only view orders with assigned to a specific `station`. Valid values are `hot`, `cold`, and `beverages`.

For example:\
`filter=station.eq~hot`

**fields (string)**\
Specify the fields to return in the response. Provide a list of field names, separated by the `|` delimiter.

At a minimum, the include the `image_name` and `image` fields in the response. For example:\
`fields=name|image_name`

### Sample Request

```json
curl -X GET "{BaseURL}/orders/698d4ef070b67b7e0cc0debd/dishes?filter=station.eq~hot&fields=name|image_name"
```

### Sample Response

```json
{
  "results": [
    {
      "id": "698d4ef070b67b7e0cc0de55",
      "name": "Risotto ai Funghi",
      "image_name": "risotto_ai_funghi"
    }
  ],
  "total_results": 1
}
```

## Display Dish Ingredients

Kitchen staff use the KDS app to retrieve a list of ingredients for the dish they are preparing. The server retrieves the ingredients and they are displayed on the KDS app.

### Endpoint

`GET /dishes/{id}/ingredients`

### Path Parameters

**id**\
Dish ID.

### Query Parameters

**fields (string)**\
Specify the fields to return in the response. Provide a list of field names, separated by the `|` delimiter.

In this case, the only relevant field for kitchen staff is the ingredient `name`:\
`fields=name`

### Sample Request

```json
curl -X GET "{BaseURL}/dishes/698d4ef070b67b7e0cc0de1f/ingredients?fields=name"
```

## Ready for Assembly

When a station completes a dish that is part of an order, the kitchen staff use the KDS app to change the dish status to `Ready for Assembly`. The client tracks when dishes are marked as `Ready for Assembly` by each station.

## Order Assembly

The expeditor assembles the dishes prepared by the stations to create the order. The expeditor then uses the KDS app to change the order status to `On the Way` (for table-side orders) or `Ready for Pickup` (for takeout orders). The server returns the order details including the updated status.

### Endpoint

`PUT /orders/{id}`

### Path Parameters

**id**\
Order ID.

### Request Body Parameter

**status**\
Order status. To mark an order as assembled and ready for the customer, set the status to `On the Way` (for table-side orders) or `Ready for Pickup` (for takeout orders).

**priority**\
Order priority. Valid values are 1-5 (1 is the highest priority).

**name**\
Patron's name.

**table\_number**\
Patron's table number

**dish\_ids**\
Array of IDs for the dishes included in the order.

### Sample Request

```json
curl -H "Accept: application/json" -H "Authorization: Bearer 4840de21-ed92-44d9-8571-fac25e81c209" -X PUT "{BaseURL}/orders/698d4ef070b67b7e0cc0dec9" -H "Content-Type: application/json" -d '{"name": "James", "table_number": 10, "dish_ids": ["698d4ef070b67b7e0cc0de94", "698d4ef070b67b7e0cc0de24"], "status": "On the Way"}'
```
